# Sistema de Achievements para Classes de Seminario

Este conjunto de documentos organiza o contexto funcional do produto em partes menores, para facilitar consulta humana e uso por AI durante design, implementacao e manutencao.

## Como navegar

- Comece por `docs/00-overview.md` para entender produto, objetivo, problema e direcao do MVP.
- Use `docs/01-roles-and-structure.md` para regras de hierarquia, perfis e estrutura organizacional.
- Use `docs/02-modules-and-flows.md` para modulos, jornadas principais e experiencia por perfil.
- Use `docs/03-achievements-and-album.md` para tudo que envolve catalogo, concessao, album e compartilhamento.
- Use `docs/04-rules-and-permissions.md` para regras formais, permissoes e auditoria.
- Use `docs/05-data-model.md` para entidades, relacionamentos e base de modelagem.
- Use `docs/06-mvp-roadmap-backlog.md` para MVP, roadmap, epicos, ordem de execucao e proximos passos.
- Use `docs/07-routes-and-screens.md` para telas sugeridas, navegacao e rotas funcionais.

- Cada arquivo tem secoes pequenas, com titulos previsiveis.
- Decisoes abertas ficam concentradas em `docs/06-mvp-roadmap-backlog.md`.
- Se surgir nova funcionalidade, adicione no arquivo tematico correspondente e atualize este indice.
